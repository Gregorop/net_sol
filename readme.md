# Тестовое: Fullstack разработчик (FastAPI + React)

Сделать небольшое приложение для учёта внутренних заявок.

# Стек

Backend: Python + FastAPI Frontend: React + TypeScript База данных: SQLite

# Модель данных

Заявка содержит следующие поля:

| Поле | Тип / правило |
| --- | --- |
| id | уникальный идентификатор |
| title | обязательное поле, от 3 до 120 символов |
| description | необязательное поле, до 1000 символов |
| status | одно из значений: new , in_progress , done |
| priority | одно из значений: low , normal , high |
| created_at | дата и время создания в UTC |
| updated_at | дата и время последнего изменения в UTC |

# Функциональные требования

Реализуйте:

1. Создание заявки.
2. Просмотр списка заявок.
3. Фильтрацию по status и priority .
4. Поиск по title и description .
5. Сортировку по дате создания и приоритету.
6. Изменение статуса заявки.
7. Удаление заявки (только админом).
8. Пагинацию списка.

# Бизнес-правила
Существует админский аккаунт с дефолтными кредами (admin:admin), в который можно зайти, админ нужен только для удаления заявок. Заявку в статусе done нельзя редактировать или удалять. Нельзя перевести заявку обратно в другой статус. При нарушении правила API должен возвращать осмысленный HTTP-ответ и понятное сообщение об ошибке. Поиск, фильтрация, сортировка и пагинация должны выполняться на backend, а не только на frontend. 

# Требования к frontend

Сделайте один экран приложения, включающий: 
* таблицу или список заявок;
* строку поиска;
* фильтры по статусу и приоритету;
* управление сортировкой;
* форму создания заявки;
* изменение статуса из списка или карточки;
* удаление заявки;
* вход в аккаунт админа;
* состояния загрузки, пустого списка и ошибок API.
Дизайн не оценивается.
